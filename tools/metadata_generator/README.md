# Steam Monitor Metadata & Protobuf Generator

This tool dynamically extracts **IPC interface vtables/signatures** and **Protocol Buffer definitions (.proto)** from compiled Steam client binaries (`steamclient64.dll` and `steamui.dll`).

It is designed to automate updating the `ipc` and `protobuf` branches of the `steam-monitor` repository when a Steam client update breaks them.

## Requirements

- Python 3.x
- The `protobuf` Python package (already installed on typical setups, or installable via `pip install protobuf`).

## Usage

You can run the script via `python generate.py` with three subcommands:

### 1. Extract both IPC and Protobuf definitions from a Steam installation
Automatically locates `steamclient64.dll` and `steamui.dll` under your Steam folder, scans them, and writes the output structured exactly for the `steam-monitor` repository branches:
```powershell
python generate.py all "C:\Program Files (x86)\Steam" .\output
```
The output directory will contain:
- `output/ipc/steamclient/<sha256>.toml` (for `ipc` branch)
- `output/protobuf/steamclient/*.proto` (for `protobuf` branch)
- `output/protobuf/steamui/*.proto` (for `protobuf` branch)

### 2. Generate IPC TOML file only
Generates vtable and method mapping TOML for the `ipc` branch:
```powershell
python generate.py ipc "C:\Program Files (x86)\Steam\steamclient64.dll" .\output_ipc
```

### 3. Extract Protobuf files only
Extracts and formats all embedded `.proto` files back into human-readable source code:
```powershell
python generate.py protobuf "C:\Program Files (x86)\Steam\steamclient64.dll" .\output_proto
```

## How It Works

### IPC Extraction
1. **Body Scan**: The tool searches the `.text` section of `steamclient64.dll` for the static 32-bit `funcHash` and `fencepost` constants generated by Valve's protocol generator for each method.
2. **Vtable Resolution**: It locates the `wrapper_rva` and checks where this address is referenced in the read-only `.rdata` section.
3. **Cross-verification**: It automatically validates candidate vtable addresses against all methods defined for the interface (`IClientUser` and `IClientUtils`) before finalizing them.

### Protobuf Extraction
1. **Dynamic Signature Matching**: Embedded serialized descriptors start with tag `0x0A` followed by the filename's length byte, and the filename string ending with `.proto`. The tool scans the entire binary to dynamically find every embedded descriptor.
2. **Minimal-Size Parsing**: Since the size is not stored inside the descriptor header itself, the tool attempts to parse the descriptor using `google.protobuf.descriptor_pb2.FileDescriptorProto` with incrementally larger slices until parsing succeeds without decode errors.
3. **Canonical Reconstruction**: It recursively traverses the parsed `FileDescriptorProto` hierarchy (services, RPC methods, messages, fields, options, and enums) and outputs standard, clean `.proto` files.
